# BuilderOS Patterns

Proven, minimal patterns. Apply the smallest one that fits — never all at once.

## Project start (always)
1. Restate the goal in one sentence and name the done-condition (a command/URL/test that proves it works).
2. Pick the smallest stack. Match the project's existing stack if one exists.
3. Create infra before features: `.gitignore`, `.env.example` (generated secrets, never hardcoded), `README.md`, one real test, linter/formatter config.

## Auth
- Hash passwords with a real KDF (PBKDF2/argon2/bcrypt). Never store plaintext or reversible hashes.
- Session tokens from a CSPRNG (`crypto.getRandomValues` / `os.urandom`), never `Math.random`.
- Secrets come from env, generated by `gh2 secrets`, and are gitignored.

## Data
- Keep persistence behind a thin repository/module boundary so storage can change without touching callers.
- For a fallback-capable client: write-through to a local store, read from it when the server is unavailable.
- Validate all external input at the boundary; treat it as untrusted.

## Frontend
- Optimistic UI where it helps; skeleton/loading states; one error-boundary/toast system.
- Mobile-first, clear hierarchy, accessible (ARIA, keyboard, contrast). See bos-frontend for taste.

## AI integration
- Free/local first (Ollama) with a graceful offline fallback; paid API only behind an optional key.
- Stdlib HTTP over heavy SDKs when it removes a dependency.
- Always validate model output before acting on it (parse, schema-check, path-check).

## Hard-won lessons
- In monolithic HTML+JS, an `onclick="fn()"` referencing an undefined `fn` crashes silently. Grep every `onclick=` and confirm each target exists before calling a task done.
- Every generated file path goes through a traversal guard (`safe_paths.py`). Reject `..`, absolute, and drive-letter paths.
